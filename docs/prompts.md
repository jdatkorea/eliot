## 2026-06-18 — BriefingView 표현 레이어 Zero-Noise (Presentation Layer Only)

```
Role: 당신은 UI 컴포넌트의 렌더링 껍데기(Presentation Layer)만 조작하여 사용자의 인지적 과부하를 없애는 프론트엔드 뷰(View) 최적화 장인입니다.

Context: 
우리는 Eliot 시스템의 백엔드, 데이터 구조, API 호출 방식, 그리고 상태 동기화 로직(CloudStorage 등)을 단 1%도 건드리지 않을 것입니다. 오직 사용자의 눈에 보이는 '표현 방식(UI/UX)'만 가볍게 수정합니다.

Task:
`app/briefing/[id]/BriefingView.tsx`의 `return` 문 내부 UI 렌더링 방식을 아래 두 가지 요구사항에 맞춰 수정하십시오. 

Action Items:
1. [표현 방식 변경 1] 듀얼 탭을 단일 텍스트 버튼으로 압축
   - 현재 화면 상단에 A와 B 버튼이 나란히 렌더링되는 탭 구조를 제거하십시오.
   - 대신 "현재: A안 · 근거리 코스 ▾" 와 같이 현재 활성화된 Variant만 보여주는 가벼운 드롭다운 스타일의 단일 버튼으로 렌더링하십시오.
   - 클릭 시 기존의 `handleVariantChange` 함수가 그대로 실행되도록만 연결하십시오. (로직 수정 금지)

2. [표현 방식 변경 2] 스왑 버튼 숨기기 (Accordian 스타일)
   - 각 브리핑 블록 우측에 항상 떠 있는 `[다른 곳으로]` 교체 버튼은 시각적 노이즈가 심합니다. 기본적으로 보이지 않게 CSS로 숨기십시오.
   - 아주 가벼운 로컬 상태(`useState`)를 하나 추가하여, 사용자가 특정 카드(Block)를 터치했을 때만 해당 카드 아래로 `[다른 곳으로 교체]` 버튼이 부드럽게 나타나도록(Expand) UI만 조작하십시오. 

Rule:
- 기존에 존재하는 함수나 비즈니스 로직(swap API 호출 등)의 내용은 절대 수정하지 마십시오.
```

**효과**: 엔진·API 무변경으로 A/B 탭→드롭다운·아코디언 스왑 완료. 회귀 반경이 `return` JSX와 로컬 `useState` 2개로 한정됨.

## 2026-06-18 — Telegram 메시지 단일 링크 (Formatting Layer Only)

```
Role: 당신은 Zero-Noise UX를 백엔드 텍스트 생성 단계까지 관철시키는 아키텍트입니다.

Context:
우리는 프론트엔드(`BriefingView.tsx`)에서 A/B 탭을 단일 드롭다운 토글로 통합했습니다. 하지만 현재 텔레그램 봇이 유저에게 보내는 결과 메시지는 여전히 A안 링크와 B안 링크를 따로따로 뱉어내고 있어 UX 불일치가 발생하고 있습니다.

Task:
텔레그램 메시지를 구성하는 백엔드 로직(예: `app/api/` 내부의 webhook 처리 파일이나 `telegram-message.ts` 등 텍스트를 포매팅하는 파일)을 찾아 아래와 같이 수정하십시오.

Action Items:
1. [단일 진입점 링크 생성]
   - 기존에 `urlA`, `urlB` 두 개를 모두 텔레그램 메시지 하단에 출력하던 로직을 제거하십시오.
   - 대신 "🔗 여정 브리핑 확인하기" 라는 단일 텍스트 링크 하나만 출력하도록 수정하십시오. 이 링크의 목적지는 기본적으로 A안(`?variant=A`)을 향하도록 세팅하십시오. (피드백 링크는 그대로 유지합니다.)

2. [안내 문구 추가]
   - 버튼이 하나로 줄어들었으므로, 메시지 본문(요약 텍스트 영역) 상단이나 하단에 "💡 웹뷰에서 두 가지 코스 옵션을 비교할 수 있습니다." 라는 짧은 안내 텍스트를 추가하십시오.

Rule:
- 데이터베이스 저장 로직이나 Supabase 구조는 절대 건드리지 마십시오. 오직 텔레그램으로 쏘아주는 '메시지 텍스트 렌더링(Formatting)' 부분만 단일 링크 구조로 변경하십시오.
- 수정된 코드 블록만 간결하게 출력하십시오.
```

**효과**: 프론트 드롭다운과 Telegram 메시지의 Zero-Noise 멘탈 모델 일치. `telegram-message.ts` 단일 파일·테스트만 수정.
